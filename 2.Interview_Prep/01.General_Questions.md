## 4. If there is issuie in prod env how will you troubleshoot ?
1. **Understand the alert** – First, I verify the alert and identify which service or application is affected.
2. **Check the impact** – I determine how many customers are affected and assess the severity of the issue.
3. **Declare the incident** – If it's a major issue, I inform the relevant teams, create an incident bridge, and keep stakeholders updated.
4. **Restore the service** – My priority is to minimize customer impact by rolling back the deployment, applying a hotfix, restarting services, or scaling resources.
5. **Find the root cause** – Once the service is stable, I analyze logs, metrics, traces, and recent deployments or configuration changes to identify the root cause.
6. **Implement the fix** – I apply the permanent fix, test it thoroughly, and deploy it through the CI/CD pipeline.
7. **Monitor and verify** – After deployment, I closely monitor the application to ensure the issue is fully resolved.
8. **Document and prevent** – Finally, I prepare the RCA, update runbooks, improve monitoring, and add preventive measures to avoid similar incidents in the future.

## 5. Mention a issuie you had faced in past and how you had troubleshoot ?
1. **Issue**: After deployment, the application pod was taking a long time to start (cold start).
2. **Impact**: The liveness probe failed before the application was ready, causing the pod to restart repeatedly.
3. **Investigation**: Checked pod status using kubectl get pods
    Reviewed pod events with kubectl describe pod
    Analyzed application logs using kubectl logs
4. **Root Cause**: The application required more time to initialize and complete startup tasks.
5. **Resolution**: Configured a Startup Probe with appropriate thresholds so Kubernetes waited for the application to become ready.
Used an Init Container to complete initialization tasks before starting the main application container.
7. **Result**: The application started successfully without unnecessary restarts, and the deployment became stable.

## 7. What KPIs Do You Track in DevOps?

* Deployment Frequency – How often code is deployed to production.
* Lead Time for Changes – Time from code commit to production deployment.
* Change Failure Rate – Percentage of deployments causing production issues.
* MTTR (Mean Time To Recovery) – Time taken to recover from incidents.
* Availability/Uptime – Service reliability and SLA compliance.

These KPIs help measure scalability, reliability, efficiency, and overall platform health.

---

## 8. What Should Be Considered During Cloud Migration?

* Application dependency mapping and assessment.
* Selection of migration strategy (Rehost, Replatform, Refactor).
* Network connectivity and hybrid architecture planning.
* Security, IAM, and compliance requirements.
* Database and data migration strategy.
* High Availability (HA) and Disaster Recovery (DR) planning.
* Monitoring, logging, and observability setup.
* Infrastructure automation using IaC.
* Cost optimization and resource right-sizing.
* Performance testing and validation before production cutover.

The goal is to ensure the migrated application is secure, scalable, highly available, operationally efficient, and cost-effective in the cloud environment.
