# Projects Interview Notes

## 1. Codegenie (AI Project)

CodeGenie is an in-house AI application that helps developers automate tasks. It reads project source code and documents, stores them in a vector database, retrieves the relevant information when needed, and uses an LLM to generate project-specific results such as unit tests, CI/CD pipelines, and code quality reports.    


1. In-house AI platform built using RAG (Retrieval-Augmented Generation).
2. Accepts source code and project documents as input.
3. Chunks the content, generates embeddings, and stores them in the Milvus vector database.
4. Users select a use case (e.g., unit test generation, CI/CD pipeline generation, code quality analysis, stored procedure to REST API conversion).
5. Users choose an LLM provider and model (e.g., Claude, Gemini, GPT, Llama).
6. Retrieves relevant context from Milvus and sends it with the user's prompt to the selected LLM.
7. Generates accurate, context-aware outputs based on the project's code and documentation.

```bash
          Source Code + Documents
                    │
                    ▼
          Parsing & Chunking
                    │
                    ▼
        Embedding Generation
                    │
                    ▼
          Milvus Vector Database
                    │
         (Semantic Retrieval)
                    │
                    ▼
      User Prompt + Retrieved Context
                    │
                    ▼
      Selected LLM (Claude/Gemini/GPT/Llama)
                    │
                    ▼
      AI-Generated Output
```

Tools used :- Python,FastAPI, LlamaIndex, Milvus Vector Database, OpenAI / Claude / Gemini (LLMs), OpenAI Embeddings, Git
