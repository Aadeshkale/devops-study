# DevOps & Cloud Interview Questions (GCP focused)

## Service-to-service communication in Cloud Run, App Engine
For Cloud Run, use the Cloud Run invoker role for calling services and use the authenticated token to request targets. Get a token using the client library or REST API. For App Engine, use App Engine with internal traffic only.

## Purpose of Shared VPC
Shared VPC allows resource sharing between multiple projects in an organization through a single VPC network by connecting VPCs. Resources from other projects can connect to the host project for internal communication.

## What is the use of Serverless VPC Access (connector)?
A VPC connector is used for connecting GCP serverless services (Cloud Run, App Engine, Cloud Functions) to a VPC to access resources using internal IP addresses. It is primarily used to access GCP resources using internal communication to send requests to the VPC and receive responses.

## How do VMs from different regions with the same VPC communicate?
Using the network interface settings in VM networking.

## Why choose App Engine (STD/FLEX) vs Cloud Run?
- Containerized applications: Use App Engine Flex or Cloud Run
- Non-containerized applications with App Engine supported languages: Use App Engine Standard
- Other factors: Serverless, cost, operations team size, manageability, need for GPU

## Why is a VM instance restarting after 24hrs (Preemptible)?
Preemptible VMs restart every 24hrs and can be taken away at any time by GCP.

## How to bring data from on-premise to cloud?
- Cloud Storage Transfer Service
- BigQuery Transfer Service
- Transfer Service based on bandwidth
- Storage Appliance

## Blue-green deployment?
Two instances of the application exist at a time: one stable (blue) and one new version with features (green). Once the new instance (service/server, green) is tested, traffic is moved from blue to green, and green becomes the new stable. Implemented with Spinnaker, Istio, ArgoCD, etc.

## Canary release deployment?
When a new version of the application is released, a small subset of traffic is moved from the stable version to the new version. After testing by a subset of users, all traffic is moved to the new version.

## What is RBAC in K8s and its use case?
RBAC is used to give role-based access control to users and pods/deployments. When a user creates a request, it goes through the API server which handles authentication, authorization, and then completes the request. If a pod/application needs access to a GKE resource, Kubernetes Service Accounts are used.

- **Role**: Set of permissions, namespace-level
- **ClusterRole**: Set of permissions, cluster-level
- **RoleBinding**: Assigns a role to a user, service account, or user group (namespace-level)
- **ClusterRoleBinding**: Assigns a ClusterRole to a user, service account, or user group (cluster-level)

## Jenkins global vs local vs user environment variables?
- **Global variables**: Can be accessed across multiple pipelines
- **Local variables**: Can be accessed within a single pipeline stage
- **User variables**: Specific to a user space

## Jenkins: how to add an approver step for a specific user?
Using the `input` step with the approver user ID.

## SonarQube quality profile and quality gate?
Every project has a quality profile set for each supported language. The profile defines which rules will be applied during analysis. After analysis, the quality gate takes the resulting metrics and compares them to its defined thresholds to determine if the code meets the requirements for release or merge.

## Cloud SQL: is it in VPC or in Google Cloud space?
Cloud SQL in private mode is created in a VPC.

## How to deploy a 3-tier application in GCP?
Put the frontend behind a load balancer. User requests come from the load balancer to the frontend (Cloud Run, MIG, Cloud Function). Put the middleware behind a load balancer that only allows traffic from the frontend load balancer. Put the database tier in a private IP space that only allows traffic from the middleware.

## How to give internet access to a GCP private VM?
Use Cloud NAT Gateway. NAT (Network Address Translation) converts private IPs to public and vice versa.

## GCP security tools?
Security Command Center for incident management, audit logs, and vulnerability management.

## What are the different types of GKE clusters?
- **Autopilot**: Supports auto-scaling of Kubernetes nodes, less flexibility on underlying networking and infrastructure
- **Standard**: More flexibility on underlying networking and infrastructure
- **Public networking**: Control plane accessible only on public networks
- **Private networking**: Control plane accessible only on private networks and authorized networks (needs GCP private access settings on subnet level for VPC-native networking)

## GCP Private Service Connect?
Allows private connectivity to managed services from your VPC without using public IPs or traversing the internet.

## GCP Workload Identity Federation?
Allows workloads running outside GCP (e.g., on-premises, AWS, Azure) to access GCP resources without service account keys, by exchanging tokens from external identity providers.

## Init vs Sidecar container?
- **Init container**: Runs and completes before the main application container starts; used for setup tasks (db migrations, config generation, permission checks)
- **Sidecar container**: Runs alongside the main container for the pod's lifetime; used for supporting tasks (logging agent, proxy, service mesh sidecar, monitoring agent)

## How to do secrets rotation?
Cloud Scheduler triggers a Cloud Function which generates new secrets, updates the database, and creates a new version in Secret Manager.

## GCP data encryption at rest, in transit, and in use?
- **Encryption at Rest**: All data stored in GCP (GCS, BigQuery, Persistent Disks, etc.) is automatically encrypted before being written to disk.
- **Encryption in Transit**: GCP automatically encrypts data in transit using TLS and other encryption protocols like IPsec.
- **Encryption in Use (In Process)**: Protects data while being processed in memory. GCP supports this with Confidential Computing using Confidential VMs and Confidential GKE Nodes.

## What are hierarchical firewall rules in GCP?
Hierarchical firewall rules allow you to apply firewall policies at the Organization, Folder, or Project level. These rules are enforced before VPC firewall rules and apply regardless of the network.
