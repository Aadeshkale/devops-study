# VPC Service Controls

* Protects Google Cloud resources from data exfiltration.
* Creates a security perimeter around one or more Google Cloud projects.
* Works in addition to IAM (does not replace it).
* Restricts access to supported Google Cloud services from outside the perimeter.
* Uses Access Context Manager to define access levels (IP, device, user, location).
* Supports Ingress (incoming) and Egress (outgoing) policies.
* Prevents copying sensitive data to unauthorized projects or networks.
* Helps protect against stolen credentials, insider threats, and misconfigurations.
* Commonly used with services like BigQuery, Google Cloud Storage, and Cloud SQL.
* Best for organizations handling sensitive or regulated data (finance, healthcare, government).
