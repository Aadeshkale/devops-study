# Helm & Helmfile Notes

## 1. What is Helm?

* Helm is a package manager for Kubernetes.
* It simplifies deployment, upgrade, rollback, and management of Kubernetes applications using Helm Charts.

---

## 2. What is a Helm Chart?

* A Helm Chart is a collection of YAML files and templates used to define Kubernetes applications.
* It contains deployments, services, ingress, configmaps, secrets, and other Kubernetes resources.

---

## 3. What are the main components of a Helm Chart?

```text
mychart/
├── Chart.yaml
├── values.yaml
├── templates/
├── charts/
└── templates/_helpers.tpl
```

### Chart.yaml

* Metadata about the chart.

### values.yaml

* Default configuration values.

### templates/

* Kubernetes manifest templates.

### charts/

* Dependency charts.

---

## 4. How do you create a Helm Chart?

```bash
helm create mychart
```

---

## 5. How do you install a Helm Chart?

```bash
helm install myrelease mychart/
```

* `myrelease` = Release Name
* `mychart` = Chart Directory

---

## 6. How do you upgrade a Helm Release?

```bash
helm upgrade myrelease mychart/
```

---

## 7. How do you list installed Helm Releases?

```bash
helm list
```

---

## 8. How do you uninstall a Helm Release?

```bash
helm uninstall myrelease
```

---

## 9. How do you pass custom values to a Helm Chart?

Using values file:

```bash
helm install myrelease mychart -f custom-values.yaml
```

Using inline values:

```bash
helm install myrelease mychart --set image.tag=latest
```

---

## 10. What is values.yaml?

* Contains default configuration values for the chart.
* Can be overridden using:

  * `-f`
  * `--set`
  * Environment-specific files

---

## 11. How do you debug Helm templates?

Render manifests locally:

```bash
helm template mychart/
```

Dry-run installation:

```bash
helm install --dry-run --debug myrelease mychart/
```

---

## 12. How do you prevent multiple Helm installs/upgrades from running simultaneously?

1. Helm has built-in release locking.
2. Use locking mechanisms in CI/CD pipelines.
3. Avoid concurrent deployments to the same release.

---

## 13. How do you add a Helm repository?

```bash
helm repo add stable https://charts.helm.sh/stable
```

---

## 14. How do you update Helm repositories?

```bash
helm repo update
```

---

## 15. How do you search for a Helm Chart?

```bash
helm search repo nginx
```

# Helmfile

## 22. What is Helmfile?

* Helmfile is a wrapper around Helm.
* Allows management of multiple Helm releases using a single declarative YAML file.

---

## 23. Why use Helmfile?

1. Manage multiple releases declaratively.
2. Environment-specific configurations.
3. Easier upgrades and rollbacks.
4. Version-controlled deployments.
5. GitOps-friendly approach.

---

## 24. Example Helmfile

```yaml
releases:
  - name: myapp
    namespace: default
    chart: stable/nginx
    values:
      - values.yaml
```

---

## 25. How do you apply a Helmfile?

```bash
helmfile apply
```

---

## 26. How do you sync Helmfile releases?

```bash

```

---

## 27. How do you list Helmfile releases?

```bash
helmfile list
```

---

## 28. How do you delete all Helmfile releases?

```bash
helmfile destroy
```

# Environments

## 29. How do you define environments in Helmfile?

```yaml
environments:
  dev:
    values:
      - dev-values.yaml

  prod:
    values:
      - prod-values.yaml
```

Deploy:

```bash
helmfile -e dev apply
helmfile -e prod apply
```

---

## 30. How do you override values in Helmfile?

```yaml
releases:
  - name: myapp
    chart: stable/nginx

    values:
      - values.yaml

    set:
      - name: image.tag
        value: latest
```

---

# Managing Multiple Environments

## 31. Approach 1 - Separate Values Files

Directory:

```text
mychart/
├── values.yaml
├── values-dev.yaml
└── values-prod.yaml
```

Deploy:

```bash
helm install myapp-dev mychart -f values-dev.yaml
helm install myapp-prod mychart -f values-prod.yaml
```

---

## 32. Approach 2 - Using --set

```bash
helm install myapp-dev mychart \
--set replicaCount=2,image.tag=dev

helm install myapp-prod mychart \
--set replicaCount=5,image.tag=prod
```

Pros:

* Simple

Cons:

* Difficult to maintain at scale

---

## 33. Approach 3 - Using Helmfile

```yaml
environments:
  dev:
    values:
      - values-dev.yaml

  prod:
    values:
      - values-prod.yaml
```

Deploy:

```bash
helmfile -e dev apply
helmfile -e prod apply
```

Recommended for enterprise environments.

---

## 34. Approach 4 - Environment Overrides

values.yaml:

```yaml
replicaCount: 2

image:
  repository: myapp
  tag: {{ .Values.env }}
```

Deploy:

```bash
helm install myapp mychart --set env=dev
helm install myapp mychart --set env=prod
```

# Helm vs Kustomize

## 35. What is Kustomize?

* Kubernetes native configuration management tool.
* Uses overlays and patches.
* Built into kubectl.

Deploy:

```bash
kubectl apply -k overlays/dev
```

---

## 36. Helm vs Kustomize

| Feature                | Helm                  | Kustomize                          |
| ---------------------- | --------------------- | ---------------------------------- |
| Purpose                | Package Manager       | YAML Customization                 |
| Templates              | Go Templates          | No Templates                       |
| Dependencies           | Supported             | Not Supported                      |
| Versioning             | Yes                   | No                                 |
| Rollback               | Yes                   | No                                 |
| Installation           | Helm CLI Required     | Built into kubectl                 |
| Environment Management | values.yaml           | Overlays                           |
| Best For               | Reusable Applications | Environment-specific Customization |

---

## 37. When should you use Helm?

1. Need templating.
2. Need package management.
3. Need dependencies.
4. Need upgrades and rollbacks.
5. Need reusable application deployments.

---

## 38. When should you use Kustomize?

1. Already have YAML manifests.
2. Need environment-specific patches.
3. Want simple configuration management.
4. Prefer Kubernetes-native tooling.

---

## 39. Can Helm and Kustomize be used together?

Yes.

Generate manifests using Helm:

```bash
helm template myapp mychart -f values-dev.yaml > output.yaml
```

Apply additional customization using Kustomize:

```bash
kubectl apply -k overlays/dev/
```

# Interview One-Liners

## 40. Helm vs Helmfile

* Helm manages a single release.
* Helmfile manages multiple Helm releases declaratively.

## 41. Helm vs Kustomize

* Helm uses templates and acts as a package manager.
* Kustomize uses overlays and patches existing YAML manifests.

## 42. Most Common Enterprise Approach

* Helm Charts for application packaging.
* Helmfile for environment and release management.
* Argo CD or Flux for GitOps deployments.
