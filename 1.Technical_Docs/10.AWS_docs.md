# Messaging Services

## SNS (Simple Notification Service)

Pub/Sub service similar to GCP Pub/Sub.

Features:

- One-to-Many Messaging
- Push-Based Delivery

Supported Targets:

- SQS
- Lambda
- Email
- SMS
- HTTP Endpoints

Architecture:

Publisher → SNS Topic → Multiple Subscribers

---

## SQS (Simple Queue Service)

Message Queue Service.

Features:

- One-to-One Communication
- Pull-Based Messaging
- Message Durability
- Decoupling Applications

Types:

### Standard Queue

- Unlimited Throughput
- At-Least-Once Delivery

### FIFO Queue

- Exactly Once Processing
- Ordered Delivery

Architecture:

Producer → SQS Queue → Consumer

# AWS Organization Structure

```text
Organization
│
├── Organizational Units (OU)
│   ├── Production Accounts
│   ├── Development Accounts
│   └── Security Accounts
│
└── AWS Accounts
```

OU is similar to GCP Folders.

AWS Accounts are similar to GCP Projects.

---

# IAM (Identity and Access Management)

## Root User

- Created during AWS Account creation.
- Has unrestricted access.
- Should not be used for daily operations.

---

## IAM User

Used for daily access.

Follows least privilege principle.

---

## IAM Role

Temporary credentials assigned to:

- Users
- Applications
- AWS Services

Examples:

- EC2
- Lambda
- ECS
- EKS

---

## Policies

Collection of permissions attached to:

- Users
- Groups
- Roles

Types:

- AWS Managed Policies
- Customer Managed Policies
- Inline Policies

---

## IAM Service Role

AWS Service assumes a role to access other AWS services.

Examples:

- CodePipeline
- Lambda
- ECS
- EKS

Equivalent to:

- GCP Service Accounts

---

# Networking

## VPC (Virtual Private Cloud)

AWS VPC is Regional.

Unlike GCP VPC, AWS VPC is not global.

---

## Public Subnet

Resources can access the internet through:

- Internet Gateway (IGW)

Examples:

- Load Balancers
- Bastion Hosts

---

## Private Subnet

No direct internet access.

Typically hosts:

- Databases
- Internal Applications
- Backend Services

Outbound internet access is provided using:

- NAT Gateway

---

# Security Groups

Instance-Level Firewall.

Features:

- Stateful
- Allow Rules Only

Default Behavior:

- All Outbound Allowed
- All Inbound Denied

Applied To:

- EC2
- Load Balancers
- RDS
- EKS ENIs

---

# Network ACL (NACL)

Subnet-Level Firewall.

Features:

- Stateless
- Allow Rules
- Deny Rules

Applied To:

- Subnets

---
