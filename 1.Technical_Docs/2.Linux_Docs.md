# Linux Notes

## 1. How do you check system uptime in Linux?

* Use the `uptime` command.

```bash
uptime
```

---

## 2. How do you find a specific process and kill it?

* Find the process using `ps` and `grep`.
* Terminate it using `kill`.

```bash
ps aux | grep process_name
kill -9 PID
```

---

## 3. What is the difference between grep, awk, and sed?

1. **grep** – Searches for patterns in text.
2. **awk** – Processes and formats text, commonly used for column extraction.
3. **sed** – Stream editor used for text substitution and transformation.

```bash
grep "error" logfile.txt
awk '{print $2, $3}' file.txt
sed 's/error/ERROR/g' file.txt
```

---

## 4. How do file permissions work in Linux?

1. **chmod** – Changes file permissions.
2. **chown** – Changes file ownership.
3. **chgrp** – Changes group ownership.

```bash
chmod 755 script.sh
chown user1 file.txt
chgrp devs file.txt
```

---

## 5. What does the umask command do?

## 9. What is the difference between curl and wget?

1. **curl** – Used to transfer data and interact with APIs.
2. **wget** – Primarily used to download files.

```bash
curl -I https://example.com
wget https://example.com/file.tar.gz
```

## 12. How do you schedule a cron job?

```bash
crontab -e
```

Example:

```bash
0 2 * * * /path/to/script.sh
```

View existing jobs:

```bash
crontab -l
```

---

## 13. How do you pass arguments to a Bash script?

* `$1`, `$2` store positional arguments.
* `$@` stores all arguments.

```bash
#!/bin/bash
echo "First argument: $1"
echo "Second argument: $2"
```

Run:

```bash
bash script.sh arg1 arg2
```

---

## 14. Where are logs stored in Linux?

1. System logs:

   * `/var/log/syslog` (Debian/Ubuntu)
   * `/var/log/messages` (RHEL/CentOS)

2. Authentication logs:

   * `/var/log/auth.log`

3. Kernel logs:

   * `/var/log/kern.log`

---


## 19. How do you use rsync for backups?

```bash

```

Options:

* `-a` Archive mode
* `-v` Verbose
* `-z` Compress data

## 24. What is $@, $*, @ and @@ in shell scripting?

1. `$@` – Represents all arguments, preserving each argument separately.
2. `$*` – Represents all arguments as a single string.
3. `@` – No special meaning by itself in Bash.
4. `@@` – No special meaning in Bash; treated as plain text.

Example:

```bash
./script.sh one two three
```

```bash
"$@" → "one" "two" "three"
"$*" → "one two three"
```

## 25. How do you run an application as a non-root user in Docker?

```dockerfile
FROM ubuntu:latest

ARG USERNAME=newuser
ARG UID=1001

RUN useradd -m -u $UID -s /bin/bash $USERNAME

RUN mkdir -p /app && \
    chown -R $USERNAME:$USERNAME /app && \
    chmod -R 770 /app

USER $USERNAME

WORKDIR /app
```

This follows container security best practices by avoiding running applications as the root user.
