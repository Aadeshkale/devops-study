# Docker Notes

## 1. What is Docker?

* Docker is an open-source containerization platform used to package, deploy, and run applications in isolated environments called containers.

---

## 2. Difference Between Containers and Virtual Machines (VMs)

1. Containers share the host OS kernel and are lightweight.
2. VMs run a complete guest OS on top of a hypervisor and are heavier.
3. Containers start in seconds, while VMs typically take longer to boot.
4. Containers consume fewer resources compared to VMs.

---

## 3. What is a Docker Image?

1. A Docker image is a lightweight, executable package containing:

   * Application code
   * Runtime
   * Libraries
   * Dependencies
   * Configuration
2. It acts as a template for creating containers.

---

## 4. What is a Docker Container?

1. A Docker container is a running instance of a Docker image.
2. Containers are isolated, portable, and lightweight.

---

## 5. How do you list running containers?

```bash
docker ps
```

List all containers:

```bash
docker ps -a
```

---

## 6. Difference Between COPY and ADD

1. **COPY**

   * Copies files from host to container.
   * Preferred for most use cases.

2. **ADD**

   * Copies files from host to container.
   * Supports downloading files from URLs.
   * Automatically extracts compressed archives.

---

## 7. What is a Dockerfile?

1. A Dockerfile is a text file containing instructions used to build a Docker image.

Example:

```dockerfile
FROM ubuntu
COPY . /app
RUN apt-get update
CMD ["python", "app.py"]
```

---

## 8. How do you build a Docker image?

```bash
docker build -t myimage .
```

---

## 9. How do you run a Docker container?

```bash
docker run -d -p 8080:80 myimage
```

---

## 10. What is Docker Compose?

1. Docker Compose is used to define and manage multi-container applications.
2. Configuration is stored in a `docker-compose.yml` file.
3. Allows multiple services to be started with a single command.

---

## 11. What are Docker Volumes?

1. Docker volumes provide persistent storage for containers.
2. Data remains available even if the container is deleted.

Example:

```bash
docker run -v myvolume:/app/data myimage
```

---

## 12. How do you remove a Docker container?

Remove stopped container:

```bash
docker rm <container_id>
```

Force remove running container:

```bash
docker rm -f <container_id>
```

---

## 13. Docker Networking Types

1. **Bridge**

   * Default network mode.
   * Communication between containers on the same host.

2. **Host**

   * Container shares host networking stack.

3. **Overlay**

   * Communication across multiple Docker hosts.

4. **None**

   * No networking enabled.

---

## 14. Difference Between CMD and ENTRYPOINT

1. **CMD**

   * Provides default command or arguments.
   * Can be overridden at runtime.

2. **ENTRYPOINT**

   * Defines the main container process.
   * Difficult to override.
   * Commonly used when the container should always run a specific application.

---

## 15. What is Docker Swarm?

1. Docker's native container orchestration platform.
2. Used to manage containers across multiple hosts.
3. Supports clustering, service discovery, scaling, and load balancing.

---

## 16. How do you scale a service in Docker Compose?

```bash
docker-compose up --scale myservice=3
```

This creates three instances of the service.

---

## 17. How do you optimize Docker images?

1. Use `.dockerignore` to exclude unnecessary files.
2. Use smaller base images such as Alpine or Distroless.
3. Minimize image layers by combining RUN commands.
4. Use multi-stage builds.
5. Remove temporary files and package caches.
6. Run applications as a non-root user.

---

## 18. Difference Between Bind Mounts and Volumes

### Bind Mounts

1. Directly map a host directory into a container.
2. Data location is controlled by the host.

Example:

```bash
docker run -v /host/data:/app/data
```

### Volumes

1. Managed by Docker.
2. Better portability and security.
3. Recommended for production workloads.

Example:

```bash
docker run -v myvolume:/app/data
```

---

## 19. How do you troubleshoot a failing container?

1. Check container logs:

```bash
docker logs <container_id>
```

2. Inspect container configuration:

```bash
docker inspect <container_id>
```

3. Access container shell:

```bash
docker exec -it <container_id> sh
```

4. Verify resource usage:

```bash
docker stats
```

---

## 20. How do you run an application as a non-root user in Docker?

```dockerfile
FROM ubuntu:latest

ARG USERNAME=newuser
ARG UID=1001

RUN useradd -m -u $UID -s /bin/bash $USERNAME

RUN mkdir -p /app && \
    chown -R $USERNAME:$USERNAME /app && \
    chmod -R 770 /app

USER $USERNAME

WORKDIR /app
```

## 21. Docker vs Podman

| Feature                | Docker                               | Podman                       |
| ---------------------- | ------------------------------------ | ---------------------------- |
| Architecture           | Client-Server                        | Daemonless                   |
| Daemon                 | Requires Docker Daemon               | No Daemon                    |
| Rootless Support       | Limited                              | Native                       |
| Security               | Daemon runs with elevated privileges | More secure by default       |
| Dockerfile Support     | Yes                                  | Yes                          |
| Registries             | Docker Hub and private registries    | Docker-compatible registries |
| Networking             | Docker networking                    | CNI-based networking         |
| Compose Support        | Docker Compose                       | Podman Compose               |
| Kubernetes Integration | Supported                            | Used by CRI-O/OpenShift      |
| Systemd Integration    | Additional configuration             | Native support               |

---

## 22. How does the Docker client communicate with the Docker daemon?

1. Docker Client sends requests to Docker Daemon through Docker REST APIs.
2. On Linux and macOS, communication usually happens through a Unix Socket:

```text
/var/run/docker.sock
```

3. For remote Docker hosts, communication can happen over TCP:

```text
tcp://host:2375
tcp://host:2376 (TLS secured)
```

4. Docker Daemon receives the request and performs actions such as building images, creating containers, managing networks, and volumes.
