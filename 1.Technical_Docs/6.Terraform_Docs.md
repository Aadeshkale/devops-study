# Terraform Notes

## 1. What is Terraform?

* Terraform is an Infrastructure as Code (IaC) tool developed by HashiCorp.
* It is used to provision and manage cloud infrastructure through code.
* Supports AWS, GCP, Azure, OpenStack, Kubernetes, and many other providers.

---

## 2. What is a Provider?

* A provider is a plugin that allows Terraform to interact with cloud platforms or services.

Examples:

* AWS
* GCP
* Azure
* Kubernetes
* OpenStack

---

# Basic Commands

## 3. terraform init

```bash
terraform init
```

* Initializes a Terraform working directory.
* Downloads providers, modules, and required dependencies.

---

## 4. terraform plan

```bash
terraform plan
```

* Shows what Terraform will create, modify, or destroy.
* Compares Terraform code with the current state file and infrastructure.

---

## 5. terraform apply

```bash
terraform apply
```

* Creates or modifies infrastructure.
* Updates the Terraform state file.

---

## 6. terraform destroy

```bash
terraform destroy
```

* Deletes infrastructure created by Terraform.

---

## 7. terraform validate

```bash
terraform validate
```

* Validates Terraform syntax and configuration.

---

# Variables

## 8. Input Variables

* Used for code reusability and environment separation.
* Avoids hardcoding values.

Default file:

```text
terraform.tfvars
```

Custom file:

```bash
terraform apply -var-file=dev.tfvars
```

---

## 9. Output Variables

* Used to expose resource attributes.
* Commonly used to pass values between modules.

Example:

```hcl
output "instance_ip" {
  value = aws_instance.web.public_ip
}
```

---

# Modules

## 10. What are Modules?

* Used for:

  * Code Reusability
  * Scalability
  * Maintainability
  * Easier Troubleshooting

Module Sources:

* Local Path
* Git Repository
* Terraform Registry

Example:

```hcl
module "vpc" {
  source = "./modules/vpc"
}
```

---

# State Management

## 11. What is a Backend?

* Used to store Terraform state remotely.

Examples:

* AWS S3
* GCS Bucket
* Azure Storage Account

Benefits:

* Centralized state management
* Prevent accidental deletion
* Team collaboration

---

## 12. State File Locking

* Prevents multiple users from modifying infrastructure simultaneously.

Examples:

* AWS S3 : now s3 also support built locking 
* Terraform Cloud
* GCS Generation-based Locking

---

# Provisioners

## 13. What are Provisioners?

Used to execute actions after resource creation.

### local-exec

Runs commands on local machine.

```hcl
provisioner "local-exec" {
  command = "echo hello"
}
```

### remote-exec

Runs commands on remote VM.

### file

Copies files from local machine to remote machine.

---

## 14. Terraform Workspaces

* Used to manage multiple environments.
* Each workspace maintains a separate state file.

Commands:

```bash
terraform workspace show

terraform workspace select dev

```

---

# Lifecycle Block

## 15. Terraform Lifecycle

Controls how Terraform creates, updates, and destroys resources.

### create_before_destroy

Creates new resource before deleting old one.

### ignore_changes

Ignores selected attribute changes.

### prevent_destroy

Prevents accidental deletion.

### replace_triggered_by

Forces recreation when another resource changes.

Example:

```hcl
lifecycle {
  create_before_destroy = true
}
```

# Interview Questions

## 16. How do you implement multiple regions?

Using provider aliases.

Example:

```hcl
provider "aws" {
  region = "us-east-1"
}

provider "aws" {
  alias  = "west"
  region = "us-west-1"
}
```

---

## 17. How do you implement multi-cloud infrastructure?

Using multiple provider blocks.

Example:

```hcl
provider "aws" {}
provider "google" {}
```

---

## 18. How do you add existing infrastructure to Terraform?

Using:

```bash
terraform import
```

or

```hcl
import {
  to = aws_instance.example
  id = "i-123456"
}
```
After this use 

```bash
terraform plan -generate-config-out=generated.tf
```
Review generated configuration


Terraformer can be used to generate Terraform code for existing resources.

---

## 19. What if a user modifies infrastructure manually?

Options:

```bash
terraform apply -refresh-only
```

or

```bash
terraform refresh
```

Best Practices:

1. Restrict console access using IAM.
2. Follow least privilege principle.
3. Enable audit logging.
4. Generate alerts for manual changes.
5. Use lifecycle ignore_changes when appropriate.

---

## 20. What is terraform taint?

```bash
terraform taint aws_instance.web
```

* Marks a resource for recreation during next apply.

---

## 21. What is Terraform Sentinel?

* Policy-as-Code framework.
* Available in Terraform Cloud and Enterprise.

Examples:

* Restrict instance types
* Enforce tagging
* Restrict public resources

---

## 22. What is a Data Block?

* Reads information about existing resources.
* Does not create resources.

Example:

```hcl
data "aws_ami" "ubuntu" {
  most_recent = true
}
```

---

## 23. What happens if Terraform CLI is interrupted?

* Terraform requests providers to stop operations safely.
* State file remains consistent whenever possible.

---

## 24. How do you migrate a state file to another backend?

```bash
terraform init -migrate-state
```

---

## 25. How do you pass data between modules?

Using outputs from one module and referencing them in another.

Example:

```hcl
module.vpc.vpc_id
```

---

## 26. How do you delete a specific resource without destroy?

Option 1:

```hcl
count = 0
```

Option 2:

```bash
terraform destroy -target=resource_name
```

---

## 27. How do you use if-else in Terraform?

Syntax:

```hcl
condition ? true_value : false_value
```

Example:

```hcl
var.env == "dev" ? var.dev_cidr : var.prod_cidr
```

---

## 28. How do you use loops in Terraform?

### count

```hcl
count = 3
```

### for_each

Creates resources dynamically.

```hcl
for_each = toset(var.storage_accounts)
```

### for expression

Transforms data.

```hcl
[for x in var.names : upper(x)]
```

---

## 29. terraform init -reconfigure vs -upgrade

### -reconfigure

```bash
terraform init -reconfigure
```

* Reconfigures backend settings.

### -upgrade

```bash
terraform init -upgrade
```

* Upgrades providers and modules.

---

## 30. What if the Terraform state file is deleted?

Recovery Options:

1. Restore from backup.
2. Restore from remote backend versioning.
3. Use terraform import to rebuild state.

Best Practice:

* Store state remotely.
* Enable bucket versioning.
* Enable backups.

---

## 31. for_each vs for expression

### for_each

* Creates multiple resources.

Example:

```hcl
resource "aws_s3_bucket" "bucket" {
  for_each = toset(var.names)
}
```

### for expression

* Transforms existing data.

Example:

```hcl
[for x in var.names : upper(x)]
```

# Interview One-Liners

## 32. Modules vs Workspaces

* Modules provide code reuse.
* Workspaces provide environment separation.

## 33. Data Block vs Resource Block

* Resource creates infrastructure.
* Data reads existing infrastructure.

## 34. Count vs for_each

* Count uses numeric indexes.
* for_each uses keys and is preferred for dynamic resources.

## 35. Most Common Enterprise Setup

* Remote State in S3/GCS
* State Locking (DynamoDB/Terraform Cloud)
* Modules for Reusability
* Separate Workspaces/Environments
* GitOps CI/CD Pipeline
* Sentinel/Policy Enforcement
