# GitLab CI & Jenkins Notes

# GitLab CI/CD

## 1. What is GitLab CI/CD?

* GitLab CI/CD is GitLab's built-in Continuous Integration and Continuous Deployment platform.
* Pipelines are defined using a `.gitlab-ci.yml` file.
* Automates build, test, security scans, and deployments.

---

## 2. What is `.gitlab-ci.yml`?

* Configuration file used to define GitLab CI/CD pipelines.
* Stored in the root of the repository.

Example:

```yaml
stages:
  - build
  - deploy

build:
  stage: build
  script:
    - echo "Building"

deploy:
  stage: deploy
  script:
    - echo "Deploying"
```

---

## 3. What are Stages?

* Logical phases of a pipeline.
* Execute sequentially.

Example:

```yaml
stages:
  - build
  - test
  - deploy
```

Common stages:

1. Build
2. Test
3. Security Scan
4. Package
5. Deploy

---

## 4. What are Jobs?

* Individual tasks executed within stages.

Example:

```yaml
build:
  stage: build
  script:
    - mvn package
```

---

## 5. What are GitLab Runners?

* Agents that execute CI/CD jobs.
* Can run on:

  * Linux VMs
  * Windows VMs
  * Docker Containers
  * Kubernetes Pods

Types:

### Shared Runner

* Used by multiple projects.

### Specific Runner

* Dedicated to a project or group.

---

## 6. What are Variables?

* Used to store reusable values.
* Can be defined:

  * Project Level
  * Group Level
  * Pipeline Level
  * Job Level

Example:

```yaml
variables:
  REGION: us-central1
```

---

## 7. What are Rules?

* Used for conditional execution of jobs.

Example:

```yaml
rules:
  - if: '$CI_COMMIT_BRANCH == "main"'
```

Common Use Cases:

* Environment selection
* Conditional deployments
* Feature branch validation

---

## 8. How do you implement branch-based deployments?

Example:

```yaml
rules:
  - if: '$CI_COMMIT_BRANCH == "main"'
    variables:
      ENV: prod

  - if: '$CI_COMMIT_BRANCH == "develop"'
    variables:
      ENV: dev
```

---

## 9. How do you share data between jobs?

Using:

### Artifacts

```yaml
artifacts:
  paths:
    - build/
```

### Cache

```yaml
cache:
  paths:
    - .m2/
```

---

## 10. What are GitLab CI/CD Artifacts?

* Files generated by one job and used by another job.
* Examples:

  * Build outputs
  * Reports
  * Test results

---

## 11. What is Cache?

* Speeds up builds by reusing dependencies.
* Examples:

  * Maven repository
  * npm packages
  * Python packages

---

## 12. GitLab CI/CD Interview One-Liner

* GitLab CI/CD uses `.gitlab-ci.yml` to define automated pipelines consisting of stages, jobs, variables, runners, and deployment workflows.

# Jenkins

## 13. What is Jenkins?

* Jenkins is an open-source CI/CD automation tool written in Java.
* Used for building, testing, and deploying applications.

---

## 14. Jenkins Architecture

Jenkins uses a Master-Agent architecture.

### Controller (Master)

Responsible for:

1. Job Scheduling
2. User Management
3. Security
4. Plugins
5. Pipeline Management
6. Tool Configuration

### Agents (Workers)

Responsible for:

1. Executing pipeline jobs
2. Running builds
3. Running tests
4. Deployments

---

## 15. What are Jenkins Agents?

* Worker machines where jobs execute.

Can be:

1. Linux VM
2. Windows VM
3. macOS VM
4. Docker Container
5. Kubernetes Pod

---

## 16. What is a Jenkins Pipeline?

* Pipeline as Code using Groovy syntax.

Stored in:

```text
Jenkinsfile
```

Benefits:

1. Version Controlled
2. Reusable
3. Automated

---

## 17. Declarative Pipeline Example

```groovy
pipeline {
  agent any

  stages {
    stage('Build') {
      steps {
        echo 'Building'
      }
    }

    stage('Deploy') {
      steps {
        echo 'Deploying'
      }
    }
  }
}
```

---

## 18. What is Multibranch Pipeline?

* Automatically creates jobs for every branch.
* Requires a Jenkinsfile in each branch.
* Automatically discovers new branches and pull requests.

---

# Jenkins Interview Questions

## 28. Jenkins vs GitLab CI

| Feature            | Jenkins      | GitLab CI         |
| ------------------ | ------------ | ----------------- |
| Installation       | Self Managed | Built into GitLab |
| Configuration      | Jenkinsfile  | .gitlab-ci.yml    |
| Language           | Groovy       | YAML              |
| Plugin Ecosystem   | Very Large   | Limited           |
| Maintenance        | Higher       | Lower             |
| Kubernetes Support | Excellent    | Excellent         |
| Scalability        | High         | High              |

---

4. Shared Libraries

# Interview One-Liners

## Jenkins

* Jenkins is a Java-based CI/CD tool that uses a Controller-Agent architecture and Pipeline as Code through Jenkinsfiles.

## GitLab CI

* GitLab CI/CD uses `.gitlab-ci.yml` files to define automated pipelines executed by GitLab Runners.

## Shared Libraries

* Shared Libraries help implement reusable pipeline code across multiple Jenkins projects.

## Multibranch Pipeline

* Multibranch Pipeline automatically discovers branches and creates Jenkins jobs for each branch containing a Jenkinsfile.
