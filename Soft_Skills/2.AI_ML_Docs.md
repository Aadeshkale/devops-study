# Local LLM Hosting & MLOps Notes

## Local Open Source LLM Hosting

### Ollama

Ollama is used to download and run open-source LLMs locally.

#### Commands

```bash
ollama list
```

List downloaded models.

```bash
ollama pull <model>
```

Download a model.

```bash
ollama run <model>
```

Run a model locally.

```bash
ollama serve
```

Start Ollama API server.

Example:

```bash
ollama pull llama3
ollama run llama3
```

## MCP (Model Context Protocol)

MCP is a protocol that allows AI models to communicate with external tools, APIs, databases, and systems.

Benefits:

- Standardized tool integration
- Dynamic context retrieval
- Secure external system access
- Agent-based workflows

Examples:

- GitHub MCP Server
- Kubernetes MCP Server
- Database MCP Server
- Filesystem MCP Server

Flow:

```text
LLM
 ↓
MCP Client
 ↓
MCP Server
 ↓
External Tool/API
```

## MLflow

Open-source MLOps platform.

### Components

#### Tracking

Track experiments and metrics.

#### Projects

Package ML code.

#### Models

Manage model formats.

#### Model Registry

Version and manage models.

### Common Functions

#### Log Parameters

```python
mlflow.log_param("learning_rate", 0.01)

mlflow.log_params({
    "learning_rate": 0.01,
    "epochs": 10
})
```

Stores values in the Parameters section.

#### Log Metrics

```python
mlflow.log_metric("accuracy", 0.95)

mlflow.log_metrics({
    "accuracy": 0.95,
    "loss": 0.02
})
```

Stores values in the Metrics section.

#### Log Artifacts

```python
mlflow.log_artifact("model.pkl")
```

Stores:

- Models
- Reports
- Images
- Datasets

## Model Serving

### KServe

KServe is a Kubernetes-native model serving platform.

Supports:

- TensorFlow
- PyTorch
- XGBoost
- Scikit-Learn
- Large Language Models (LLMs)

### Components Created by KServe

When a model is deployed, KServe automatically creates:

- Deployment
- Service
- HPA (Horizontal Pod Autoscaler)
- Ingress / Gateway
- Model Runtime

### InferenceService

Main CRD used for model deployment.

Example:

```yaml
apiVersion: serving.kserve.io/v1beta1
kind: InferenceService
metadata:
  name: sklearn-model
spec:
  predictor:
    sklearn:
      storageUri: gs://models/sklearn
```

### Benefits

- Serverless scaling
- Autoscaling to zero
- Canary deployments
- GPU support
- Multi-model serving
- LLM serving support
- Kubernetes native

## Interview Question

### What is the difference between MLflow and KServe?

#### MLflow

Used for:

- Experiment Tracking
- Model Registry
- Model Lifecycle Management

#### KServe

Used for:

- Model Deployment
- Model Serving
- Autoscaling
- Production Inference

**Simple Answer:**

MLflow manages and tracks models, while KServe deploys and serves models in production.
