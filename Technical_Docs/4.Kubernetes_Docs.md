# Kubernetes Notes (Recommended Order)

## 1. Pod

* Smallest deployable unit in Kubernetes.
* Can contain one or more containers sharing storage and networking.

## 2. ReplicaSet

* Maintains the desired number of pod replicas.
* Uses label selectors.
* Provides high availability.

## 3. Deployment

* Manages ReplicaSets and Pods.
* Supports rolling updates and rollbacks.
* Recommended for stateless applications.

## 4. StatefulSet

* Used for stateful applications such as MySQL, PostgreSQL, MongoDB, Kafka.
* Creates pods in order, if pods get deleleted it recreated with same name and same storage attached to it .
* Provides stable DNS names and dedicated persistent storage for pods.

## 5. DaemonSet

* Runs one pod on every node.
* Commonly used for logging and monitoring agents.

Examples:

* Fluentd
* Datadog Agent
* Prometheus Node Exporter

---

# Services

## 6. ClusterIP

* Internal communication only.
* Accessible within the cluster.

## 7. NodePort

* Exposes application on a port of every node.

## 8. LoadBalancer

* Creates a cloud load balancer trough cloud resource manager.
* Exposes application using a public IP.

## 9. Headless Service

* Created using:

```yaml
clusterIP: None
```

* Direct communication with pod IPs.
* Commonly used with StatefulSets.

Example:

```text
mysql-0.mysql.default.svc.cluster.local
mysql-1.mysql.default.svc.cluster.local
```

# Ingress & Gateway

## 10. Ingress

* Single entry point for multiple applications.
* Routes traffic using:

  * Domains
  * Subdomains
  * Paths

### Types

#### External Ingress

* Public traffic.
* GKE ingress class: gce

#### Internal Ingress

* Internal traffic.
* GKE ingress class: gce-internal

#### Multi-Cluster Ingress

* Routes traffic across multiple clusters.

#### NGINX Ingress

* Third-party ingress controller.

---

## 11. Gateway API

* Future replacement for Ingress.
* More modular and flexible.
* Uses:

  * GatewayClass
  * Gateway
  * HTTPRoute
  * TCPRoute

# Health Checks

## 12. Readiness Probe

* Checks whether pod is ready to receive traffic.
* Failed pod is removed from service endpoints.

## 13. Liveness Probe

* Checks whether application is healthy.
* Failed pod is restarted.

## 14. Startup Probe

* Used for slow-starting applications.
* Prevents premature liveness failures.

# RBAC

## 28. RBAC

* Role-Based Access Control.
* Controls access to Kubernetes resources.

Request Flow:

```text
User → API Server → Authentication → Authorization → Resource
```

## 29. Role

* Namespace-level permissions.

## 30. ClusterRole

* Cluster-wide permissions.

## 31. RoleBinding

* Assigns Role to User, Group, or Service Account.

## 32. ClusterRoleBinding

* Assigns ClusterRole cluster-wide.

# Scheduling

## 34. Node Selector

* Uses exact key:value matching.
* Simple scheduling mechanism.

## 35. Node Affinity

* Advanced version of Node Selector.
* Supports:

  * Required rules
  * Preferred rules

## 36. Pod Affinity

* Schedules pods close to other pods.

## 37. Pod Anti-Affinity

* Prevents pods from running on the same node.
* Improves high availability.

## 38. Taints and Tolerations

* Taints repel pods from nodes.
* Tolerations allow pods onto tainted nodes.

# Node Maintenance

## 54. Node Cordon

```bash
kubectl cordon node-name
```

1. Marks node as Unschedulable.
2. Existing pods continue running.
3. New pods are not scheduled on the node.

---

## 55. Node Drain

```bash
kubectl drain node-name --ignore-daemonsets
```

1. Evicts workload pods from the node.
2. Scheduler moves pods to other nodes.
3. Used during node upgrades, patching, and maintenance.
