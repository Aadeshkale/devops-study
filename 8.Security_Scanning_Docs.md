# Security Scanning Notes

## 1. What is SAST (Static Application Security Testing)?

* SAST scans source code, binaries, or artifacts without running the application.
* Identifies security vulnerabilities early in the development lifecycle.
* Typically integrated into CI/CD pipelines.

Common Findings:

* Hardcoded secrets
* SQL Injection risks
* Cross-Site Scripting (XSS)
* Insecure coding patterns
* Dependency vulnerabilities

### Popular SAST Tools

1. SonarQube
2. Snyk Code
3. Checkmarx
4. Fortify
5. Semgrep
6. Veracode

---

## 2. What is DAST (Dynamic Application Security Testing)?

* DAST scans a running application.
* Simulates real-world attacks against deployed applications.
* Does not require source code access.

Common Findings:

* SQL Injection
* Cross-Site Scripting (XSS)
* Authentication Issues
* Authorization Issues
* Security Misconfigurations

### Popular DAST Tools

1. OWASP ZAP
2. Burp Suite
3. Qualys WAS
4. Acunetix
5. Invicti (Netsparker)

---

## 3. SAST vs DAST

| Feature                | SAST        | DAST               |
| ---------------------- | ----------- | ------------------ |
| Testing Type           | Static      | Dynamic            |
| Source Code Required   | Yes         | No                 |
| Application Running    | No          | Yes                |
| SDLC Phase             | Development | Testing/Production |
| Detects Coding Issues  | Yes         | No                 |
| Detects Runtime Issues | No          | Yes                |

# Container Security

## 4. What is Container Image Scanning?

* Scans container images for:

  * Vulnerabilities
  * Misconfigurations
  * Secrets
  * Malware
  * License Issues

Usually integrated into:

* CI/CD Pipelines
* Container Registries
* Kubernetes Admission Controls

---

## 5. Free Container Security Scanners

### Trivy

* Most popular open-source scanner.
* Scans:

  * Container Images
  * Filesystems
  * Git Repositories
  * Kubernetes Clusters
  * IaC

Example:

```bash
trivy image nginx:latest
```

---

### Clair

* Open-source container vulnerability scanner.
* Often integrated with container registries.

---

### Grype

* Open-source vulnerability scanner from Anchore.

Example:

```bash
grype nginx:latest
```

---

### ThreatMapper

* Open-source cloud-native security platform.
* Scans:

  * Containers
  * Kubernetes
  * Hosts
  * Cloud workloads

---

# Software Composition Analysis (SCA)

## 6. What is Dependency Scanning?

* Identifies vulnerabilities in third-party libraries and packages.
* Detects known CVEs in dependencies.

Examples:

* Maven Packages
* npm Packages
* Python Packages
* Go Modules

---

## 7. Free Dependency Scanners

### OWASP Dependency Check

* Identifies vulnerable third-party libraries.

Example:

```bash
dependency-check.sh --scan .
```

---

### Snyk Open Source

* Dependency vulnerability scanning.

---

### Horusec

* Open-source security analysis platform.
* Supports multiple languages and frameworks.

---

# Cloud Security

## 8. What is CSPM (Cloud Security Posture Management)?

* Continuously scans cloud environments for:

  * Misconfigurations
  * Compliance Violations
  * Security Risks

Examples:

* Public Buckets
* Open Security Groups
* Excessive IAM Permissions

# Common Security Pipeline

## 10. Security Scanning in CI/CD

Typical Pipeline Flow:

```text
Developer Commit
        ↓
SAST Scan (SonarQube/Semgrep/Snyk)
        ↓
Dependency Scan (OWASP Dependency Check)
        ↓
Build Container Image
        ↓
Container Scan (Trivy/Grype)
        ↓
Deploy to Test Environment
        ↓
DAST Scan (OWASP ZAP)
        ↓
Deploy to Production
```

# Interview Questions

## 11. When should SAST be performed?

* During development and build stages.
* Before application deployment.

---

## 12. When should DAST be performed?

* After application deployment to a test or staging environment.
* Requires a running application.

---

## 13. Which free container scanner do you prefer?

* Trivy is the most commonly used open-source container scanner due to its simplicity, speed, and Kubernetes/IaC scanning capabilities.

---

## 14. What security scans do you integrate into CI/CD?

1. SAST (SonarQube, Semgrep, Snyk)
2. Dependency Scan (OWASP Dependency Check)
3. Container Scan (Trivy)
4. Secrets Scan (Gitleaks, TruffleHog)
5. DAST (OWASP ZAP)

---
